# Day 02 — Profiles vs Agents vs Skills

## Терминология (уточнение после обсуждения)

| Понятие | Что это | Реализация в Claude Code | Аналог (Starcraft) |
|---------|---------|--------------------------|-------------------|
| **Профиль** | Глобальная настройка СЕССИИ — меняет поведение Клода на весь диалог | `/mode-bug-fix` slash-команда, `.claude/rules/`, CLAUDE.md | Режим игры |
| **Агент** | Изолированный процесс под конкретную ЗАДАЧУ — получает вход, возвращает результат | `.claude/agents/*.md` | Юнит |
| **Скилл** | Переиспользуемая команда/абилка | `.claude/commands/*.md` (=slash-команды) | Абилка юнита |

**Ключевое отличие:**
- Профиль живёт всю сессию, меняет "личность" ассистента
- Агент спавнится под задачу, работает изолированно, возвращает результат и "умирает"
- Скилл — команда, которую можно вызвать в любой момент

## Что реализовано в Day 02

### Профили (slash-команды, `.claude/commands/`)

Входят в сессию через `/mode-bug-fix` или `/mode-research` и остаются активными:

#### `/mode-bug-fix`
Глобальный режим отладки для всей сессии. Клод ведёт себя как senior KMP-разработчик в режиме баг-хантинга: любой запрос воспринимается как описание бага.

**MUST:** greps весь проект → читает файл целиком → описывает root cause до правки → проверяет все файлы того же типа  
**MUST NOT:** рефакторинг, новые абстракции, правки без Read  
**Формат:** `## Что нашёл / ## Что починил / ## Что проверил`

#### `/mode-research`
Глобальный режим аналитика (read-only). Клод отвечает только структурированными отчётами, ничего не меняет.

**MUST:** Glob → Read → Grep → строить явные связи файл:строка  
**MUST NOT:** Edit, предложения фиксов, утверждения без кода  
**Формат:** `## Вопрос / ## Исследованные файлы / ## Связи / ## Выводы / ## Пробелы`

---

### Агенты (`.claude/agents/`, исключены из git через `.git/info/exclude`)

Спавнятся через `Agent` tool для изолированных задач:

#### `bug-fixer`
Инструменты: `Read, Glob, Grep, Edit`  
Получает: описание бага → возвращает: отчёт + правки

#### `researcher`
Инструменты: `Read, Glob, Grep`  
Получает: вопрос → возвращает: структурированный анализ

#### `smart-day-builder`
Инструменты: `Read, Glob, Grep, Edit, Write, Bash`  
Расширение `kmp-day-builder` с фазой анализа предыдущего дня + предложением механики

## Итерации (что дорабатывали)

### Профиль Bug Fix
- Добавлено: проверять ВСЕ файлы того же типа, не только сообщённый
- Добавлено: раздел "Что проверил" обязателен

### Профиль Research
- Добавлено: запрет утверждать без строки кода (галлюцинации)
- Добавлено: требование указывать `файл:строка`, не только пакеты

### Уточнение терминологии (после обсуждения участников)
- Первичная реализация путала профиль и агент
- Исправлено: профили → slash-команды (`/mode-*`), агенты → `.claude/agents/`
- Подход продвинутых участников подтверждён: профили в командах, CLAUDE.md — только базовые паттерны
